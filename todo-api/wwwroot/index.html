Some helpers

  • You can use swagger to test and play around with the C# API. Use this to your advantage, for example you can use it to populate the backend with data before implementing the view.
  • At any point you should be able to view the existing items at this url.
  • We're using bootstrap as a base CSS! It comes with some basic styles and the ability to make cool layouts! You can choose a different theme here, just replace the href in the linked stylesheet in the header.
  • Use the internet to your advantage! Many people run into the same problems, this means you should be able to find awnsers to common questions. Stack Overflow was built exactly for this! Google might also have awnsers for you. Articles on W3Schools are a good starting point for web development and .NET has good documentation over at Microsoft.

Requirements

  • JS AJAX Display all existing todos.
  • JS AJAX Ability to add new todo.
  • JS AJAX User should be able to edit existing todo.
  • JS AJAX Todos can be deleted.
  • JS Display confirmation popup when clicking on delete.
  • JS Show the time since todo was last edited. Like 54 minutes ago.
  • JS AJAX C# Allow the user to reorder the todo items.
    • you need to extend the C# API for this
  • C# Todo list should not dissapear when reloading applications. Store the todo items in a file.
    • Reimplement ITodoService in a new class. Don't change the existing one! You should always be able to switch back or to another!
  • C# NuGet Persist the todo items in a database.
    • MySQL recommended (it's free) but anything goes!
    • Use the Dapper NuGet package!
